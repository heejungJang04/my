# UMC — 대학생 연합 IT 동아리 React 팀 프로젝트

> 협업 경험 대표 사례 (자소서 4번 문항 / 면접 단골)

## 상황
- 팀원 전원이 개발 초보였음 (나 포함)
- 레포 생성·기본 구조 세팅조차 막막, 조장도 노베이스
- 자칫 프로젝트가 시작도 못 하고 멈출 뻔함

## 내가 한 일
- 조장은 아니었지만 **먼저 나서서 폴더 구조·개발 환경 등 기본 베이스 세팅을 도맡음**
  (※ 레포 '생성'은 조장이 함 → "기본 베이스 세팅"으로 정직하게 표현)
- 조장과 서로 도우며 팀 출발 토대 마련
- **말수 적던 팀원이 '로그인 기능' 구현에서 막힘 → 곁에 붙어 함께 코드 보며 해결**
- 첫 단추가 끼워지자 팀 전체가 속도 냄

## 느낀 점
- 협업은 가장 잘하는 사람이 이끄는 게 아니라, 막혔을 때 **먼저 손을 내미는 사람**에게서 시작된다
- 완벽하지 않아도 먼저 시도하고 동료에게 곁을 내주는 태도가 팀을 움직인다

## 자소서/면접 활용 포인트
- **협업·주도성**: 조장 아닌데 먼저 나섬 → 스타트업·공기업 모두 호감
- ⚠️ 면접 대비: 그 팀원이 로그인에서 정확히 뭐가 막혔는지 한 단어라도 정해둘 것
  (예: API 연동 / 상태관리 / 토큰 저장 / 에러 처리)
