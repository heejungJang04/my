# HIRA 위암 환자 담석증 위험 예측 + 자가관리 앱

> 공공기관(HIRA) 제공 의료데이터 기반 팀 프로젝트 (5명, 단기 공모전)
> **내 역할: 데이터 분석 + 서비스 기획 + Figma 화면 설계** (코딩 X)

## 한 일 (요약)
- 문제의식: 위 절제술 환자는 담낭 수술까지 가면 위험이 크게 높아짐 → 미리 예측·관리하자
- 데이터: 명세서·진료·상병·원외처방 내역에서 위암→위절제술→담석 환자를 단계적으로 추출
  - 전체 약 1,500여 명 → 위암 25명 → 위암+담석 3명 → 위절제술 13명 → 위암+수술 9명
  - 최종 위암+수술+담석 동시 발병 = 극소수(사실상 1명)
- 예측 모델링: 시도했으나 **양성 사례가 너무 적어 데이터 불균형 심함 → SMOTE 증강에도 성능 안 나옴(실패)**
- 보완: 외부 논문·심평원 연구로 교차검증 (위절제술 후 담석 위험 약 2배, 5년후 5.2%)
- 약물 검증: UDCA(이담제)가 예방 효과 → 데이터에서 약가코드→주성분코드→약효분류번호 추적해 처방 확인
- 서비스 기획: '담석증 자가관리 앱' (상태 점수화 / 복약·검사 알림 / 수술 시기별 커뮤니티), Figma로 화면 설계

## 배운 점
- 데이터의 양·품질이 결과를 좌우한다
- 작은 데이터에 모델을 억지로 맞추기보다 도메인 지식·외부 근거와 결합하는 게 낫다
- 분석은 분석으로 끝나는 게 아니라 사용자에게 닿는 서비스로 이어질 때 가치가 있다

## 자소서/면접 활용 포인트
- **데이터 직무**: 분석·전처리·불균형 처리(SMOTE)·교차검증 경험
- **공공성**: 공공 의료데이터로 사람의 삶에 닿는 경험 (SH·여성가족재단에 강함)
- **정직성**: "모델은 실패했지만 정직하게 다루고 다른 근거로 보완" → 면접에서 호감
- ⚠️ 면접 대비: 내가 직접 한 분석 범위(추출/전처리/SMOTE/약가코드)를 한 문장으로 정해둘 것
