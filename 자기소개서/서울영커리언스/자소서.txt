[문항 1] 지원 직무의 업무내용 및 핵심역할과 지원 이유  (약 600자)
───────────────────────────────────────────────────────────

프론트엔드 개발은 디자인을 화면으로 옮기는 일이 아니라, 사용자의 행동과 서버의 데이터가 만나는 접점을 설계하는 일이라고 생각합니다. 버튼 하나에도 로딩·에러·중복 요청 같은 수많은 상태가 얽혀 있고, 이 상태를 어떻게 다루느냐가 서비스의 완성도를 결정하기 때문입니다.

제가 React와 TypeScript를 선택한 이유도 여기에 있습니다. React는 화면을 상태(state) 단위로 쪼개어 복잡한 인터랙션을 예측 가능하게 관리할 수 있고, TypeScript는 서버와 주고받는 데이터 구조를 타입으로 고정해 협업 중 생기는 오류를 미리 잡아주기 때문입니다. 실제로 게시판 댓글 기능을 만들 때, 댓글·대댓글의 응답 구조를 interface로 정의하고 등록·삭제 시 얽히는 여러 상태를 직접 제어하면서 이 두 기술이 왜 협업과 유지보수에 강한지 체감했습니다.

저는 여기서 한 걸음 더 나아가, 만든 화면을 데이터로 검증하고 개선하는 개발자가 되고 싶습니다. 화면 너머의 사용자 데이터를 읽어 다음 기능을 제안할 수 있을 때 프론트엔드가 진짜 가치를 가진다고 믿습니다. 제 프론트엔드 구현력과 데이터 분석 경험을 잇는 이 직무에 지원했습니다.


[문항 2] 지원 직무와 관련한 본인의 역량 또는 경험  (약 1,050자 → 1,000자로 축약 필요)
───────────────────────────────────────────────────────────

저는 '직접 만드는 힘(개발)'과 '데이터로 판단하는 힘(분석)' 두 축을 갖추고 있습니다.

[개발] 동아리 플랫폼을 React·TypeScript로 개발하며 지식공유 게시판의 댓글 시스템을 맡았습니다. TypeScript를 택한 이유는 게시글·댓글의 API 응답 구조를 interface로 고정해 런타임 오류와 협업 중 데이터 형태 혼선을 줄이기 위해서였습니다. 가장 신경 쓴 건 비동기 처리였습니다. 댓글 등록 API 응답 전 사용자가 버튼을 다시 누르면 중복 댓글이 생기는 문제를, isSubmitting 상태 플래그 + early return 가드 + 버튼 비활성화 + try/finally 복구 패턴으로 막았습니다. 댓글 수는 등록 시 서버 재조회로 정합성을 보장하되, 삭제·대댓글에서는 setState 함수형 업데이트(prev => prev.count - 1)로 낙관적 반영해 응답 속도를 높였습니다. 또 사용자 입력은 dompurify로 sanitize 후 렌더링해 XSS를 차단했고, 마지막 댓글 삭제로 빈 페이지가 되면 이전 페이지로 자동 이동시키는 등 엣지 케이스까지 처리했습니다.

[데이터] 공공 의료데이터(HIRA)로 위암 환자의 담석증 위험을 분석할 땐, 명세서·진료·상병·원외처방 4개 파일을 수진자 코드로 조인하고 분류·항목 코드로 위절제술 환자를 식별했습니다. 그러나 위암·수술·담석을 모두 가진 양성 표본이 1,553명 중 1명뿐이라 SMOTE 오버샘플링으로도 성능 확보가 불가능했습니다. 저는 멈추지 않고 약가코드→주성분코드→약효분류번호를 역추적해 데이터 속 UDCA(이담제) 처방을 찾아냈고, 심평원 논문과 교차검증(위절제 후 담석 발생 약 2배)으로 타당성을 입증한 뒤 자가관리 서비스로 기획해 수상했습니다.

저는 이렇게 기술 선택의 '왜'를 설명할 수 있는 지원자입니다.


[문항 3] 사업 참여기간 동안 이루고 싶은 목표와 향후 발전계획  (약 650자)
───────────────────────────────────────────────────────────

저는 React·TypeScript로 웹 서비스를 만들고 Vercel로 직접 배포하며, 기획부터 화면 구현·배포까지 이어지는 흐름을 경험해 왔습니다. 또 공공 데이터(HIRA)를 분석해 인사이트를 도출하는 데이터 역량도 함께 키웠습니다. 이제는 이 경험을 실무 환경에서 한 단계 더 깊이 있게 발전시키고 싶습니다.

인턴십에서의 목표는, 실제 사용자가 쓰는 서비스의 코드와 데이터를 다루며 현업의 개발·협업 방식을 빠르게 흡수하는 것입니다. 학생 프로젝트와 달리 실무는 규모 있는 트래픽과 운영 중인 실데이터, 체계적인 코드 리뷰가 있는 만큼, 제가 가진 프론트엔드 구현력과 데이터 분석 경험을 바로 현장에 보태면서 더 견고하고 확장 가능한 개발을 익히고 싶습니다.

나아가 새로운 기술 흐름에도 열려 있는 개발자가 되고자 합니다. 최근에는 LLM·MCP가 서비스에 결합되는 흐름에 관심을 갖고 공부하고 있으며, 변화에 빠르게 적응해 데이터를 근거로 더 나은 기능을 제안하고 구현하는 개발자로 성장하겠습니다.
