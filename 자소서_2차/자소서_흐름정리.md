# 서울영커리언스 자소서 — 흐름 & 전략 정리

> 2026 서울영커리언스 인턴십(가을학기) / 직무: IT·개발·데이터
> 이 문서는 자소서가 "어떤 전략과 흐름으로 구성됐는지" 한눈에 보기 위한 가이드입니다.

---

## 1. 지원 개요
- **지원 구성**: 1순위 더블에이스(스타트업) / 2순위 SH(공기업) / 3순위 여성가족재단(연구·엔지니어링)
- **핵심 전략**: 자소서는 **한 편**으로 세 기업에 동시 지원됨 → 특정 회사를 못박지 않고 **직무(IT·개발·데이터) 중심 + 중립 톤**으로 작성
- **글자수**: 각 문항 500~1,000자(공백 포함) — ① 837 / ② 679 / ③ 598 / ④ 702 전부 통과
- **블라인드**: 출신지역·학교명 미기재 (학과만 허용)

## 2. 전체 컨셉 — 한 줄 정체성
> **"화면을 만드는 손(프론트엔드) + 데이터를 읽는 눈(데이터)"** 을 모두 갖춘, 두 영역을 잇는 개발자

- 추상적 표현("열정·성장")을 배제하고, **실제 코드·경험 기반 디테일**로 작성
  (isSubmitting·dompurify·페이지 경계 보정·누수 제거·약가코드 추적·GCP 등)

## 3. 문항별 흐름

| 문항 | 흐름 | 핵심 |
|------|------|------|
| **① 지원동기** | 프론트엔드를 "사용자 행동과 데이터의 접점 설계"로 정의 → React·TS 선택 이유(상태·타입 통제 = 신뢰의 토대) → 만들고 배포하며 얻은 관점("동작하는 화면 ≠ 신뢰할 수 있는 화면") → 프론트+데이터 강점으로 지원 | 직무관 + 기술 선택 철학 + 강점 |
| **② 역량** | 개발(동아리 플랫폼 댓글 기능: 중복 방지·정합성·XSS·UI/UX) + 데이터(딥페이크 데이터 생성·GCP, 캐글 대회 누수 제거·익명화 설계) | 만드는 힘 + 데이터 다루는 힘 (코드 디테일) |
| **③ 목표** | 실무에 빠르게 적응 + 새 기술 학습 적응력 + AI 코딩 도구(Claude Code·Copilot) 활용 역량 | 성장 방향 + AI 활용 (직무 중립) |
| **④ 협업** | 무박 5일 공모전, 모델 실패 위기 → 약가코드로 분석 타당성 입증 → 방향 전환 조율 → 수상 | 위기에서 근거로 팀 방향 전환 |

## 4. 프로젝트 배치 (문항 간 중복 없이 분산)
- **BAMBOO**(동아리 플랫폼) → ② 개발
- **ReGen-DF**(딥페이크 데이터셋) → ② 데이터 생성
- **캐글 대회**(게임 인기도 예측) → ② 데이터 설계
- **HIRA**(위암 담석 예측, 수상작) → ④ 협업

→ 한 프로젝트를 여러 문항에서 반복하지 않고, 각 문항이 다른 경험을 보여주도록 배치

## 5. 핵심 메시지 (면접에서도 일관되게)
1. **만들기 + 데이터, 둘 다 한다** (프론트엔드 구현 + 데이터 분석/생성/설계)
2. **기능을 넘어** 사용자 경험·데이터 정합성·보안까지 고려한다
3. **위기에서** 감정이 아니라 **객관적 근거로 방향을 바꾼다**
